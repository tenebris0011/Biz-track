# Biz Track

A self-hosted business expense and mileage tracker built for freelancers. Track income and expenses, log mileage with automatic round-trip distance calculation, and export everything to CSV at tax time.

## Features

- **Transactions** — Log one-off income and expenses with categories and notes
- **Recurring rules** — Set up repeating income/expenses (subscriptions, retainers, rent, etc.)
- **Mileage tracking** — Log trips with automatic round-trip distance via OpenStreetMap + OSRM
- **Yearly overview** — P&L summary, monthly bar chart, and IRS Schedule C category breakdown
- **CSV import** — Import transactions or trips from a CSV with flexible column mapping and saved templates
- **CSV export** — Export transactions or trips filtered by date range
- **Categories** — Custom income/expense categories managed from settings
- **Auth** — Single-user email/password login (sign-up disabled; create your account via the seed script)

## Getting Started

### Prerequisites

- Node.js 20+ (via [nvm](https://github.com/nvm-sh/nvm) recommended)
- npm

### Local development

```bash
# Install dependencies
make install

# Copy and configure environment
cp .env.example .env
# Edit .env — set DATABASE_URL, BETTER_AUTH_SECRET, BETTER_AUTH_URL, NEXT_PUBLIC_APP_URL

# Apply database migrations
make migrate

# Start dev server
make dev
```

Open [http://localhost:3000](http://localhost:3000).

### Environment variables

| Variable | Description | Example |
|---|---|---|
| `DATABASE_URL` | Path to the SQLite database file | `/app/data/biz-track.db` |
| `BETTER_AUTH_SECRET` | Random secret for session signing | `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | Full public URL of the app | `https://biztrack.example.com` |
| `NEXT_PUBLIC_APP_URL` | Same as above (used client-side) | `https://biztrack.example.com` |

### Creating your user account

Sign-up is disabled in the UI. The easiest path is to temporarily enable it in `src/lib/auth.ts` (`disableSignUp: false`), register your account at `/auth/signin`, then set it back to `true` and redeploy.

## Deployment (Docker + Traefik)

The app ships as a Docker container and expects a running Traefik instance on an external `proxy` network.

```bash
# Build and start
make deploy

# View logs
make docker-logs

# Stop
make docker-down
```

The `docker-compose.yml` is pre-configured for Traefik with:
- Host rule: `biztrack.home.wilsoncode.space`
- Entrypoint: `websecure` (HTTPS)
- TLS cert resolver: `cloudflare`
- Middleware: `secure-headers@file`

Edit the labels in `docker-compose.yml` to change the domain.
